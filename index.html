Resumo

VRML é uma linguagem para modelagem de mundo virtuais em 3D. Da mesma forma que o HTML dá suporte à construção de páginas WEB o VRML dá suporte a criação de mundos em três dimensões, que são acessiveis pelo próprio navegador.

Criada em 1994, o VRML tem o objetivo de aproximar a realidade virtual da internet. Foi um padrão aberto que proveu uma fácil adaptabilidade, além de ter sido desenvolvido, primariamente, para fins educativos e experimentos.

VRML também é conhecido como Virtual Reality Markup Language

Introdução

VRML utiliza texto para definir as credenciais dos objetos 3D, como os itens de coordenadas 3D e os valores geometricos são especificados e transformados na ilustração ou animação original.

  • Utilização no mundo

    Um arquivo VRML contém a descrição do mundo, onde ontem textos que podem ser criados em qualquer editor ou processador. A expressão mundo é utilizada para referenciar os arquivos com extensão ".wrl", pois quando o navegador lê o arquivo, ele constroi o "mundo" descrito no arquivo.

  • Ambientes
  • A linguagem

    VRML obriga a presença de um cabeçalho. O corpo do arquivo é composto por nodos que descrevem figuras contidas nele e as suas propriedades no mundo virtual. O nodo é a estrutura sintática básica de composição deste tipo de arquivo. Tipicamente, contem o tipo do nodo, um par de chaves e um conjunto de campos com seus atributos.

    Exemplo:

    
Cylinder{
	height 2.0
	radius 2.0
}

    Nesse exemplo, Cylinder é o tipo de nodo e height e radius são seus chaves e valores da altura e diâmetro do cilindro a ser desenhado

    Existe um tipo pré-definido de tipo de dados que são aceitos como atributo de um campo no nodo. Alguns desses tipos são: SFBool, SDColor, SFImage, SFInt32, SDNode, SFString, SFTime, SFVec2f.

    Os nodos presentes em um mundo podem ser nominados. A definição de um nome para um nodo tem o objetivo de identificá-lo para uso posterior, a definição de um nome é feita da seguinte mandeira:

    
DEF nome_do_nodo tipo_do_nodo {...}

    Outro exemplo: 

    
DEF Meu_Cilindro Cylinder{
	height 2.0
	radius 2.0
}

    Para descrever uma figura é preciso descrever uma forma geometrica. Além disso, esta figura tem sua aparência baseada no materia com o qual ela é feita. Dessa forma, geometry e apearance são campos do nodo Shape.

    
Shape {
	Appearance DEF Azul Appearance { # Campo da aparência da figura
		material Material {
			diffuseColor 0.0 0.0 1.0
		}
	}

geometry Cylinder { # Definição da sua forma 3D
		height 2.0
		radius 2.0
	}
}

Material {
	ambienteIntensity 0.2 # SFFloat
	diffuseColor 0.8 0.8 0.8 # SFColor
	emissiveColor 0.0 0.0 0.0 # SFColor
	shininess 0.2 # SFFloat
	specularColor 0.0 0.0 0.0 # SFColor
	transparency 0.0 # SFFLoat
}
